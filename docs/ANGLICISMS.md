# Lista anglicyzmów w pracy

Poniżej zestawiono anglicyzmy używane w tekście pracy wraz z proponowanymi tłumaczeniami lub konwencjami. Decyzja co do każdego z nich wymaga ustalenia.

| Anglicyzm (forma w tekście) | Przykład użycia | Proponowane tłumaczenie / uwaga |
|---|---|---|
| **prompt** / **promptów** / **promptowania** | „sposoby formułowania zapytań (promptów)" | Brak dobrego polskiego odpowiednika; rozważ: *instrukcja*, *zapytanie* — ale w obszarze LLM termin jest de facto standardem. Można zostawić z definicją przy pierwszym użyciu. |
| **prompt engineering** / **prompt engineeringu** | „techniki inżynierii promptów" | W tekście pojawia się obie formy — „inżynieria promptów" i „prompt engineering". Warto wybrać jedną konwencję i stosować konsekwentnie. |
| **prompt injection** | „ataki typu prompt injection" | Brak standardowego tłumaczenia; sugestia: *wstrzyknięcie instrukcji* lub zostawić w oryginale z cudzysłowem przy pierwszym użyciu. |
| **pipeline** / **pipeline'u** | „potoku walidacyjnego (*pipeline*)" | Sugerowane tłumaczenie: **potok** (już używane w tekście). Sprawdzić, czy wszystkie wystąpienia są przetłumaczone. |
| **fine-tuning** / **fine-tuningu** | „bez dodatkowego etapu dostrajania (fine-tuningu)" | Sugerowane tłumaczenie: **dostrajanie** (już używane w tekście). Sprawdzić spójność. |
| **tool calling** / **function calling** | „funkcje *tool calling*" | Brak tłumaczenia w tekście. Propozycja: *wywołania narzędzi* lub *wywoływanie narzędzi* — już stosowane w diagramie. |
| **benchmark** / **benchmarki** | „benchmark CloudEval-YAML" | Sugerowane tłumaczenie: **punkt odniesienia** lub **test porównawczy**, ale „benchmark" jest powszechnie rozumiany. Można zostawić jako termin techniczny. |
| **rollback** | „mechanizm rollback" | Sugerowane tłumaczenie: **wycofanie zmian** lub **przywrócenie stanu**. |
| **human-in-the-loop** | „podejście human-in-the-loop" | Sugerowane tłumaczenie: **człowiek w pętli** lub **nadzór ludzki w procesie**. |
| **token** / **tokenów** | „sekwencji tokenów" | Standardowy termin w NLP — można zostawić; przy pierwszym użyciu: *żeton (token)*. |
| **endpoint** / **endpointów** | „testowych endpointów HTTP" | Sugerowane tłumaczenie: **punkt końcowy** — częściowo używane w tekście. Sprawdzić spójność. |
| **seed** | „parametr seed" | Sugerowane tłumaczenie: **ziarno losowości** — częściowo używane w tekście. Sprawdzić spójność. |
| **linting** / **lintingu** / **lintera** | „narzędzie do lintingu" | Sugerowane tłumaczenie: **statyczna analiza** — już używane w tekście. Sprawdzić, czy wszystkie wystąpienia są ujednolicone. |
| **tag** / **tagu** | „tagu latest" | Sugerowane tłumaczenie: **znacznik** lub zostawić — termin bardzo ugruntowany w ekosystemie Docker. |
| **patch** / **patcha** | „aplikuje patch w formacie unified diff" | Zmienione w rozdziale 4 na: *łatka (patch)*. Sprawdzić pozostałe wystąpienia. |
| **diff ratio** | „metryka diff ratio" | Można tłumaczyć jako **współczynnik różnicy** lub zostawić jako termin techniczny z definicją. |
| **best practices** / **best practises** | „checklistę dobrych praktyk" | Używane naprzemiennie — ujednolicić do **dobrych praktyk** lub zostawić oryginalny termin w tytule wariantu promptu. |
| **zero-shot / few-shot** | „tryb zero-shot" | Brak polskich odpowiedników; można zostawić jako terminy techniczne. |
| **chain-of-thought (CoT)** | „technika CoT" | Brak polskiego odpowiednika; skrót CoT jest zdefiniowany w wykazie skrótów — OK. |
| **runtime** | „walidacja runtime" | Sugerowane tłumaczenie: **środowisko uruchomieniowe** lub **etap uruchomienia** — używane w tekście naprzemiennie. Sprawdzić spójność. |
